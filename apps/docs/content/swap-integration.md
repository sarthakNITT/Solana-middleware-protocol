Liquidity swaps are the most time-sensitive transactions on Solana. During high-volatility events, standard submission frequently fails due to rapid price movements (slippage) or network congestion. This guide explains how to use Sendra to land high-certainty swaps.

## 1. The Challenges of Swapping

When building a swapping interface, you face two competing pressures:
- **Price Drift**: The target price moves between the moment you get a quote and the moment the block is produced.
- **Inclusion Conflict**: Thousands of bots are contending for the same liquidity pool writable accounts.

## 2. Integrating with Jupiter API

Sendra is the perfect companion for the Jupiter Aggregator. By using Sendra's `built` mode, you can wrap the complex transactions generated by Jupiter.

```typescript
// 1. Get the swap transaction from Jupiter
const { swapTransaction } = await (await fetch('https://quote-api.jup.ag/v6/swap', { ... })).json();

// 2. Pass it to Sendra for high-reliability execution
const result = await SendWithReliability(
  { 
    type: "built", 
    transaction: VersionedTransaction.deserialize(Buffer.from(swapTransaction, 'base64')) 
  },
  wallet
);
```

## 3. The Role of Pre-flight Guard in Swaps

Sendra's mandatory simulation (Stage 4) is critical for swaps. If the price moves too far and the swap would fail its "Slippage Check" on-chain, Sendra's simulation will detect this immediately. 

- **Benefit**: Instead of broadcasting an invalid transaction and paying priority fees for a failed land, Sendra stops the execution and returns a logical error. This saves the user SOL and provides an instant opportunity to refresh the quote.

## 4. Optimization for Flash Sales & TGEs

During Token Generation Events (TGEs), the network is saturated. Sendra overcomes this by:
- **Provider Rotation**: Automatically switching from a congested Helius node to a private cluster tip.
- **Aggressive Fee Scaling**: Escalating priority fees on retries to ensure the swap remains competitive against bot traffic.

## Developer Insight: Commitment Choice

For swaps, the choice between `confirmed` and `finalized` is important. For 99% of traders, `confirmed` is perfect. If you are a high-frequency shop where "finalized" is required for accounting, be aware that Sendra will continue the retry loop until the transaction is fully rooted in the cluster's majority stake, which takes ~31.2 seconds in ideal conditions.
